# Git Practice Lab

This repository is a hands-on workspace for practicing essential Git commands in a real development environment. It is used to build confidence with version control workflows such as staging, committing, branching, merging, and pushing changes to a remote repository.

## 🎯 Purpose

The goal of this project is to strengthen practical Git skills, including:

- Initializing repositories
- Tracking file changes
- Staging and committing updates
- Working with branches
- Pushing and pulling from GitHub
- Resolving basic merge workflows

## 🛠️ Git Commands Practiced

### Initialize a repository
```bash
git init
```

### Check repository status
```bash
git status
```

### Add files to staging
```bash
git add .
```

### Commit changes
```bash
git commit -m "your commit message"
```

### Connect to remote repository
```bash
git remote add origin <repo-url>
```

### Push changes to GitHub
```bash
git push -u origin main
```

### Pull latest changes
```bash
git pull origin main
```

### Create and switch branches
```bash
git checkout -b feature-branch
```

### Merge branches
```bash
git merge feature-branch
```


## 💡 What This Helps With

- Understanding how Git tracks changes
- Getting comfortable with real-world workflows
- Building muscle memory for DevOps and engineering tasks
- Preparing for collaborative development environments
